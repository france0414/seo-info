為什麼設計師需要懂技術 SEO？

SEO 的底層邏輯與系統無關（不管是 Odoo 還是 Next.js 都適用）。當我們在規劃跨國網站時，搜尋引擎不僅在閱讀我們的內容，還在理解我們的「結構」與「對應關係」。

給設計師的一句話：

「良好的設計應該讓搜尋引擎像使用者一樣，能輕鬆找到正宗且正確語言的頁面。」

301 轉址：網頁的「搬家告示」

當我們改版、更改網址路徑或下架舊頁面時，Google 的紀錄裡仍存有舊網址。如果什麼都不做，使用者進來會看到 404。

錯誤：直接丟棄舊網址 (404)

就像店面拆了卻沒掛告示。Google 會認為這家店倒了，累積多年的關鍵字權重會歸零。

正確：設定 301 轉址

就像在舊店門口掛上「本店已遷至新址」的紅布條，權重會順利轉移到新頁面。

🎨 設計師任務：

  • 只要網址動過，就必須列出「舊對新網址清單」。
  • 確保新頁面內容能對應舊頁面需求，不讓跳出率提高。

Canonical Tag：網頁的「正宗商標」

當網站因為篩選功能（顏色、排序）或廣告追蹤而產生多個網址，但內容卻是大同小時，我們需要聲明誰才是「正傳」。

example.com/product-a
example.com/product-a?color=blue
example.com/product-a?utm_source=fb
Canonical URL
example.com/product-a
⚠️ 注意：跨站點自指

在跨國站點中，台灣站頁面必須指回台灣站自己，不可全部指向首頁。這確保各站點在當地都有獨立的權力。

「設計師應確保在 Metadata 規範中，『規範網址』是每個頁面的必填屬性。」

Hreflang：給爬蟲看的「地圖導航」

這標籤告訴 Google：「這頁是給繁中用戶看，那頁是給英文用戶看」。如果沒設，Google 可能會讓台灣人在搜尋結果中看到英文站。

🇹🇼
hreflang="zh-TW"
.../zh-tw/home
🇺🇸
hreflang="en-US"
.../en/home
🌍
hreflang="x-default"
.../en/home

# 什麼是 x-default？

這是「降落傘」頁面。當使用者的語言/地區不在我們名單內（例如法國使用者造訪），我們應該引導他去的頁面（通常是全球英文版首頁）。

技術禁忌：不要強制自動 IP 跳轉

❌ SEO 災難：自動跳轉 (Auto-Redirect)

許多設計師希望「自動」讓台灣人去台灣站。但 Google 爬蟲大多來自美國 IP，一旦設定自動跳轉，爬蟲會被永遠鎖在英文站，抓不到台灣站的內容。

🤖
Google Crawler
Access Denied!

✅ 最優解：智慧引導提示 (Prompt)

當偵測到使用者 IP 與當下站點不符時，彈出一個美觀的視窗提示建議切換，但允許使用者關閉並繼續閱讀當下內容。
